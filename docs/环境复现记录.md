# openpi 环境复现记录

## 环境

| 项 | 值 |
|---|---|
| 系统 | Linux (Ubuntu) |
| 项目路径 | `/mnt/cpk/magic/openpi` |
| Python | 3.11.11 |
| uv | 0.11.26 |
| 代理 | `http://192.168.32.28:18000` |

## 复现步骤

```bash
# 1. 进入项目
cd /mnt/cpk/magic/openpi

# 2. 拉取子模块
git submodule update --init --recursive

# 3. 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh
source $HOME/.local/bin/env

# 4. 设置代理（需翻墙时）
export http_proxy=http://192.168.32.28:18000
export https_proxy=http://192.168.32.28:18000

# 5. 安装依赖
GIT_LFS_SKIP_SMUDGE=1 uv sync
GIT_LFS_SKIP_SMUDGE=1 uv pip install -e .

# 6. 激活虚拟环境
source .venv/bin/activate
```

## 遇到的问题

| 问题 | 原因 | 处理 |
|---|---|---|
| `bash: uv: 未找到命令` | 安装后 `~/.local/bin` 未加入 PATH | 执行 `source $HOME/.local/bin/env` |
| 下载超时/失败 | 网络需代理 | 设置 `http_proxy` / `https_proxy` |
| `Failed to acquire lock` | 多个 `uv sync` 同时运行 | 结束残留 uv 进程后重试 |

## 结果

- `uv sync` 成功，240 个包已就绪
- `uv pip install -e .` 成功，`openpi==0.1.0` 已以 editable 模式安装
- 虚拟环境：`.venv/`
- `Failed to hardlink files` 为警告，不影响使用（缓存与目标目录不在同一文件系统）

环境安装完成，可执行 `source .venv/bin/activate` 后使用。
